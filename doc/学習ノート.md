## RDBとの比較

| 観点 | RDB | DynamoDB |
|------|-----|----------|
| スキーマ | 固定（事前定義） | 柔軟（スキーマレス） |
| スケーリング | 垂直（サーバー増強） | 水平（自動分散） |
| クエリ | SQL（柔軟なJOIN） | Key-Value + Query |
| トランザクション | ACID完全サポート | 制限付きサポート |
| 設計アプローチ | データ中心 | アクセスパターン中心 |
| コスト | サーバー常時稼働 | 使用量課金 |

### DynamoDBが向いているケース
- 読み書きが高頻度で低レイテンシが必要
- 水平スケーリングが必要
- アクセスパターンが事前に明確
- 単純なKey-Valueアクセスが中心

### RDBが向いているケース
- 複雑なJOINやアドホッククエリが多い
- 強い整合性が常に必要
- アクセスパターンが不明確・変化しやすい
